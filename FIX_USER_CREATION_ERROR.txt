🔍 WHAT'S THE PROBLEM?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

When Supabase creates a user, it runs a trigger to auto-create a profile.

The old trigger was trying to create a profile WITHOUT a company_id.
Since company_id is required, it failed.

Solution: Make company_id optional so profiles can be created first,
then assign the company later.
