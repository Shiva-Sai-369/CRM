ROLE SELECTOR
  Why: Users must identify as Admin or Worker before login
  How: Two side-by-side buttons, one becomes blue when selected
  Effect: Active button grows slightly and gets shadow for visual feedback

PASSWORD SHOW/HIDE
  Why: Users can verify password before submitting
  How: Eye icon on right side of password field
  Effect: Click to toggle between bullets (•••) and plain text

LOADING STATES
  Why: Prevents duplicate submissions and shows action is processing
  How: Button text changes to "Signing in..." with spinner
  Effect: Button becomes disabled until action completes

ERROR DISPLAY
  Why: Users need clear feedback if login fails
  How: Red box appears above form with error message
  Effect: User can read exact error and try again

FORGOT PASSWORD
  Why: Users might forget their password
  How: "Forgot password?" link opens modal
  Effect: User enters email, system sends reset link

GOOGLE OAUTH
  Why: Quick login without creating new password
  How: "Continue with Google" button
  Effect: Click redirects to Google login, then back to app
